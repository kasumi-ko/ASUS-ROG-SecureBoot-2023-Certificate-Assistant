# Security Policy

## Sensitive files

Never upload or attach any of the following to a public issue, discussion, forum, or cloud link:

- repair workflow recovery packages;
- raw `PKDefault.bin`, `KEKDefault.bin`, `dbDefault.bin`, or `dbxDefault.bin` backups;
- BitLocker recovery keys;
- device serial numbers;
- private logs that have not been reviewed.

Use the sanitized diagnostic report generated by the application when reporting a normal bug.

## Reporting a security issue

Use GitHub private vulnerability reporting if it is enabled for the repository. Otherwise, open a minimal public issue that does not include exploit details or sensitive files and ask the maintainer for a private contact channel.

## Supported version

Security fixes are applied to the latest published release.
